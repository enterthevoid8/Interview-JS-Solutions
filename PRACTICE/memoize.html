🧠 Memoization in JavaScript

Cache function results to avoid expensive recalculations

📖 What is Memoization?

Memoization is an optimization technique that stores the results of expensive function calls and returns the cached result when the same inputs occur again.

Key Benefits:

  • Prevents redundant calculations
  • Improves performance for expensive operations
  • Trades memory for speed (space-time tradeoff)

1️⃣ Basic Memoization Implementation

// Simple memoization function function memoize(fn) { const cache = {}; return function(...args) { const key = JSON.stringify(args); if (key in cache) { return cache[key]; // Return cached result } const result = fn(...args); cache[key] = result; // Store in cache return result; }; }

Try it: Square Function

2️⃣ Classic Example: Fibonacci Numbers

// Without memoization - exponential time O(2^n) function fibonacci(n) { if (n <= 1) return n; return fibonacci(n - 1) + fibonacci(n - 2); } // With memoization - linear time O(n) const fibMemo = memoize(fibonacci);

3️⃣ Memoizing Async Functions (API Calls)

// Memoize async functions function memoizeAsync(fn) { const cache = {}; return async function(...args) { const key = JSON.stringify(args); if (key in cache) { return cache[key]; } const result = await fn(...args); cache[key] = result; return result; }; }

4️⃣ Advanced: Memoization with TTL (Time To Live)

// Memoization with expiration time function memoizeWithTTL(fn, ttl = 5000) { const cache = {}; return function(...args) { const key = JSON.stringify(args); const now = Date.now(); if (cache[key] && now - cache[key].timestamp < ttl) { return cache[key].value; } const result = fn(...args); cache[key] = { value: result, timestamp: now }; return result; }; }

5️⃣ LRU Cache (Least Recently Used)

// LRU Cache with max size function memoizeWithLRU(fn, maxSize = 3) { const cache = new Map(); return function(...args) { const key = JSON.stringify(args); if (cache.has(key)) { const value = cache.get(key); cache.delete(key); cache.set(key, value); // Move to end (most recent) return value; } const result = fn(...args); cache.set(key, result); if (cache.size > maxSize) { const firstKey = cache.keys().next().value; cache.delete(firstKey); // Remove oldest } return result; }; }

// React.memo - memoizes component const ExpensiveComponent = React.memo(function({ data }) { return <div>{heavyCalculation(data)}</div>; }); // useMemo - memoizes values const memoizedValue = useMemo(() => { return expensiveCalculation(a, b); }, [a, b]); // useCallback - memoizes functions const memoizedCallback = useCallback(() => { doSomething(a, b); }, [a, b]);

✅ When to Use Memoization

✅ Good Use Cases:

  • Expensive calculations (recursion, math)
  • Repeated API calls with same params
  • Pure functions (same input → same output)
  • Data transformations
  • Search/filter operations

❌ Avoid When:

  • Functions are already fast
  • Results are rarely reused
  • Memory is constrained
  • Functions have side effects
  • Inputs change frequently
